ABSTRACT

In this project, a prototype and implementation of Mood Angels, a comprehensive mental health application, is demonstrated. The proposed system consists of an interactive user interface built with React and Tailwind CSS, and a robust server-side architecture powered by Node.js, Express, and MongoDB. Mood Angels aims to help users track their daily Mood , manage wellness through daily check-ins, and seamlessly connect with professional psychiatrists. It allows users to maintain a secure medical history encompassing diagnoses, treatments, medications, and symptoms.

A significant feature of this application is the integration of an AI insights engine, developed using Python and Scikit-learn. The Machine Learning model analyzes users' logged data over time to provide personalized wellness recommendations and Mood pattern visualization. The platform also offers secure authentication (including Google OAuth) and appointment scheduling capabilities. This system serves as a scalable, responsive, and highly effective digital healthcare solution that bridges the gap between individuals seeking mental health support and medical professionals.

CHAPTER - 1

1. INTRODUCTION

1.1 Purpose of the project

The primary purpose of Mood Angels is to provide a unified platform for mental health monitoring and professional consultation. Mental well-being is an essential aspect of overall health, and maintaining consistent records of Mood variations, sleep patterns, and energy levels can significantly aid in diagnosing and treating mental health issues. This application aims to make Mood tracking intuitive while also bridging the gap between users and certified psychiatrists via a built-in scheduling system and AI-powered insights.

1.2 Problem with Existing Systems

Existing mental health platforms often face several limitations:

  • Fragmentation: Users typically have to use separate applications for Mood tracking, journaling, and appointment scheduling with professionals.
  • Lack of Actionable Insights: Standard Mood trackers only provide basic graphs but fail to analyze underlying patterns or offer AI-driven recommendations based on historical data.
  • Poor Patient-Doctor Connection: Many platforms do not allow users to easily share their medical history or tracked symptoms securely with a registered psychiatrist prior to an appointment.
  • Complex User Interfaces: A complicated UI can deter individuals who are already struggling with mental health from consistently logging their data.

1.3 Proposed System

The proposed system, Mood Angels, offers an all-in-one comprehensive solution. It features an interactive dashboard where users can easily log their daily Mood sleep quality, and physical discomfort. The system utilizes a Node.js/Express backend to securely store this data in a MongoDB database. A distinct feature of this system is its integration with a Python-based Machine Learning pipeline that analyzes user data to generate personalized AI insights. Furthermore, the platform incorporates a dedicated module for psychiatrists, enabling them to register, manage their availability, and conduct scheduled sessions with users.

1.4 Scope of the Project

The scope of Mood Angels includes:

  • Developing a secure authentication mechanism supporting both standard credentials (bcrypt-hashed) and Google OAuth.
  • Creating separate, customized dashboard experiences for regular users and psychiatrists.
  • Implementing a daily check-in feature to record Mood energy, and symptoms.
  • Building an ML model to interpret logged data and identify potential mental health patterns or warnings.
  • Facilitating an appointment booking system that seamlessly connects users with professionals.

CHAPTER - 2

2. LITERATURE SURVEY

The rise of digital healthcare and telepsychiatry has driven significant research into mobile and web-based mental health interventions. Studies show that Ecological Momentary Assessment (EMA)—the real-time logging of Mood and behaviors—drastically improves the accuracy of patient data compared to retrospective reporting during clinical visits.

Historically, applications like Daylio or Mood ath popularized digital Mood tracking, but they often function as standalone journals. Recent literature emphasizes the integration of Artificial Intelligence (AI) in predicting depressive episodes by analyzing subtle changes in daily inputs (e.g., disrupted sleep combined with low energy). The implementation of Scikit-learn models in healthcare applications provides a lightweight yet powerful method to generate these predictive insights. Mood Angels builds upon these foundations by not only incorporating predictive AI features but also providing a direct portal for psychiatric intervention, effectively closing the loop between assessment and treatment.

3. SOFTWARE REQUIREMENT SPECIFICATION

3.1 Introduction to SRS

This Software Requirement Specification (SRS) outlines the functional and non-functional requirements for the Mood Angels application. It serves as a guide for the development team and provides stakeholders with a clear understanding of the system's capabilities.

3.2 Role of SRS

The SRS acts as the fundamental blueprint for Mood Angels, detailing the technology stack, system constraints, user interactions, and specific goals like integrating the Python ML component with the Node.js backend.

3.3 Requirements Specification Document

The application targets two primary user roles: the general user (patient) and the psychiatrist. The specification details the workflows specific to each role.

3.4 Functional Requirements

  • Authentication: The system shall allow users to sign up and log in using email/password or Google OAuth.
  • Mood Tracking: Users shall be able to log daily check-ins (Mood sleep, energy).
  • AI Insights: The system shall process user history using an ML model to return personalized wellness tips.
  • Medical Profile: Users shall be able to update their medical history, current medications, and symptoms.
  • Appointment System: Psychiatrists shall be able to set their availability, and users shall be able to book, view, and manage appointments.

3.5 Non-Functional Requirements

  • Security: All passwords must be hashed using bcryptjs. User sessions must be managed securely using JWT tokens or secure session cookies.
  • Responsiveness: The React interface must be fully responsive and optimized for mobile and desktop views using Tailwind CSS.
  • Scalability: The MongoDB schema should be designed to handle scaling as the number of user logs increases over time.

3.6 Performance Requirements

The application should load the dashboard in under 2 seconds. The Python ML inference should execute and return insights within 3 seconds to maintain a seamless user experience.

3.7 Software Requirements

  • Frontend: React 19, Vite, Tailwind CSS, React Router DOM.
  • Backend: Node.js, Express.js, Passport.js.
  • Database: MongoDB (Mongoose ORM).
  • Machine Learning: Python 3.x, Pandas, Scikit-learn, Joblib.

3.8 Hardware Requirements

Any modern device (PC, smartphone, tablet) with a standard web browser and an active internet connection. The server hosting the application requires a minimum of 2GB RAM and a modern multi-core CPU to handle Node server requests and Python ML execution efficiently.

4.3 Technologies Used

The project leverages the MERN stack integrated with Python for analytics. React enables dynamic, component-based UI rendering. Tailwind CSS provides utility-first rapid styling. Node.js with Express handles routing, middleware, and API endpoints. MongoDB provides a flexible NoSQL structure ideal for unstructured daily check-in logs. Scikit-learn is utilized to train and deploy the mental health predictive model.

CHAPTER - 5

5. IMPLEMENTATION

5.1 Setting up Database and Backend Server

The implementation began with initializing a Node.js server and connecting to MongoDB using Mongoose. Schemas for User, Psychiatrist, Daily Check-in, and Appointment were defined. Secure routing was implemented using Passport.js for Google OAuth and JWT for traditional authentication. Environment variables were utilized to secure sensitive connection strings and secrets.

5.2 Coding the Core Logic & AI Insights

A Python script (`train_model.py`) was developed using Scikit-learn to train a classification/regression model on historical Mood data. The trained model was serialized using Joblib. In the Node.js backend, the `python-shell` library was integrated to execute inference scripts dynamically, passing the user's recent check-in data as arguments and parsing the standard output to retrieve the AI-generated recommendations.

5.3 Connecting Frontend to Backend API

The frontend was bootstrapped with Vite. Components for the Dashboard, Profile, and Appointment booking were constructed using React functional components and hooks (`useState`, `useEffect`). Axios/Fetch API was used to interact with the backend REST endpoints. Tailwind CSS was applied globally to ensure a modern, clean, and accessible UI.

CONCLUSION

The Mood Angels project successfully delivers a comprehensive mental health application that modernizes how individuals monitor their well-being and seek professional help. By integrating advanced web development frameworks (React, Node.js) with Machine Learning (Python, Scikit-learn), the platform transcends a simple digital journal. It provides actionable AI insights, secure medical history management, and a robust appointment scheduling ecosystem. Mood Angels demonstrates the profound potential of technology in promoting mental wellness and making psychiatric care more accessible.
